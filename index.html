记一次群辉NAS的iSCSI LUN故障

Posted on 2021-02-09 | In default
Words count in article: | Reading time ≈

背景

服务端通过iscsi协议挂在一个群辉设备的iSCSI卷。并把这个盘作为共享文件，客户端通过Samba协议连接到服务器，并将文件传输落盘至这个iSCSI卷。

故障现象

  1. 客户端传输速度变慢，多数时候为0kb/s;
  2. 当有传输任务发起时，客户端打开映射盘时会出现无响应的情况；
  3. 传输任务容易中断。

原因分析

  1. 网络原因1：客户端到服务端的网络链路。
  2. 网络原因2：服务器端到存储设备的网络链路。
  3. 服务器原因1：服务器存在异常进程或者服务。
  4. 服务器原因2：服务器负责较高。
  5. 存储服务器原因1：设备存在故障，如故障硬盘、阵列降级。
  6. 存储服务器原因2：所在的iSCSI LUN存在异常。

Windows-动态访问控制

Posted on 2021-01-31 | In Windows Server
Words count in article: | Reading time ≈

DAC动态访问控制

文件的访问控制是基于 NTFS 文件系统来执行的。因此要使用此特性，文件系统必须为NTFS。

NTFS 特性

  1. 节约空间
  2. 磁盘压缩
  3. 更加安全
  4. EFS加密
  5. 磁盘配额
  6. 卷影副本
  7. 单个文件最大支持64GB

访问控制

操作流程：

  1. DC中启用足额策略：计算机管理-系统-KDC-KDC支持声明、复合身份验证和Kerberos Armoring
  2. 声明属性:在动态访问控制中，创建声明，如department 、title
  3. 创建规则：在动态访问控制中，进入 Central Access Rules中，创建规则
  4. 创建策略：在动态访问控制中，进入 Central Access Policy 中添加规则
  5. 目标文件服务器，选中对应文件夹的属性，高级，中央策略，选择对应Policy

Claim Types 声明
Central Access Rules 规则
Central Access Policy 策略（包含多个规则）

Policy 通过组策略发布，在客户端生效。

“拒绝优先”

ACL列表中，依次从上至下执行，被列为拒绝权限的用户，优先执行。

需求场景

创建一个public的文件夹，仅允许公司内的主管访问。

传统做法

创建一个安全组，将主管的AD拖进同一个安全组，然后再对Public进行授权。

Windows活动目录学习

Posted on 2021-01-31 | In Windows Server
Words count in article: | Reading time ≈

基础概念

Item Detail
域 用来描述一种系统架构，和“工作组”相对应，由工作组升级二来的高级架构，在域架构中，可以实现统一化管理。
活动目录 微软提供的目录服务器（查询、身份验证），活动目录的核心包含了活动目录数据库，在活动目录数据库中包含了域中的所有对象（用户，计算机，组）
域控制器 在域架构中用来管理客户端的服务器，是域架构的核心，每个域控制器上包含了活动目录数据库。

组策略

Question：组策略中的设置“计算机管理”和“用户管理”管理对象分别是什么？
Answer：

  • 计算机管理，对应的是活动目录下的计算机
  • 用户管理，对应的是活动目录下的用户

DC部署流程

  1. 创建Windows Server，将服务器加入现有的域。（如果当前不存在域控，可忽略）
  2. 安装域控相关的服务器。（Active Directory 、 Domain Server等）
  3. 将服务器提升为域控制器。

注：

  1. 作为域控的服务器文件系统需要为NTFS。
  2. 不需要提前安装DNS服务，因为安装过程中会自动安装DNS服务。

域控问题

【功能级别】林功能级别和域功能级别是什么意思？

  • “林”功能级别，控制的是整个林中的域控制器的操作系统版本。
  • “域”功能级别，控制的是整个域中的域控制器的操作系统版本。

【域名申请】公网域名为rohman.cn ， 域名要为rohman.cn还是rohman.local？

如果域名使用rohman.cn ，公网上对外发布的服务器域名为crm.rohman.cn时，内网需要单独添加解析，否则会无法查询到此记录。

关于 netlogon 、 sysvol

sysvol ： 组策略模板文件脚本文件。
netlog

关于域是一个安全边界

Domain Detail
rohman.cn 父域
pz.rohman.cn 为rohman.cn的子域
sg.rohman.cn 为rohman.cn的子域

子域与父域是信任关系。

父域的管理员 rohman.cn/rohman.zhu 无法管理两个子域，仅有读取权限。

关于域是管理边界

关于域是一个复制单元

  1. 域中的DC的数据库是相互复制的（目的是为了确保多个副本）
  2. 同一个站点中，默认15s复制一次。不同站点的复制频率则取决于站点链路的频率。
  3. 复制类型有两种：多主复制、单主复制。

活动目录数据库有三个逻辑分区：Domain ， Configuration ， Schema ；同一个域中的DC，三个分区都会数据同步；如果不是同一个域，则只同步Configuration 、 Schema分区。

目录树与目录林

目录树类似于一个内家庭不同家庭成员分支。
目录林类似于一个家族内的不同家庭分支。

物理结构

站点表示的是一个物理区域。

站点

好处：可以让当地的服务器寻找当地的DC去验证。只有在本地站点出现故障的情况下，才会去其他站点做验证。

【站点同步】

默认触发：默认情况下，站点是180分钟发起一次同步。
手动触发：“Active Domain站点和服务” 选中对应站点的NTDS Setting选中链路，右键-立即复制

验证成功的前提条件：

  1. 输入DC中存在的账户，并且是正确的账户与密码。
  2. DC中能找到当前登录的计算机。

文件服务器

DFS（Distributed File System）分布式文件系统

Failover Clust

主要用于后端存储服务器（SQL Server，Exchange Mailbox ，FS）

前提条件：

  1. 必须要域架构
  2. 标准的Cluster需要的共享存储（SQL Server的 Always on）
  3. 仲裁服务器（服务器总数为偶数，则需要仲裁服务器。因为偶数个服务器无法裁决集群中的服务器健康状态，需要有投票节点。）

群集的两种模式：

  1. Active/Passive:主备模式
  2. Active/Active:双主模式

部署流程

  1. 安装DFS命名空间、DFS复制
  2. 分别在两个文件夹中创建相同名字的文件夹
  3. 创建文件索引，同一个索引指向两台服务器
  4. 使两个文件夹相互复制

DFS文件服务器，最小容错单位是文件夹。

PC的主频

Posted on 2021-01-10 | In 服务器与存储 , 服务器
Words count in article: | Reading time ≈

频率

倍频与主频，主频就是CPU的时钟频率；倍频即主频与外频之比的倍数。主频、外频、倍频，其关系式：主频＝外频×倍频。

外频

外频是CPU乃至整个计算机系统的基准频率，单位是MHz（兆赫兹）

PC 平台的基础频率,100 MHz （Base Clock）

PC内的所有设备根据这个基础频率工作。

倍频

随着技术进步，CPU的运算速度要远高于内存、硬盘灯其他设备，而由于其他产商的技术进步并未达到CPU的速度。因此CPU既保持着外频跟着其他设备进行通讯，又存在倍频是自身能够全速运行。

同样内存也有倍频，例如内存的主频是2400HMz，则倍频是24.

内存外频异步工作

历史原因，内存的频率可以比外频高33.33333的量级去运行。为了实现这个需求，会让内存存在不同的外频去运行，即内存速度转化比率Ratio（例如 100：133 或 100：100）。

因此内存主频会有：

  • 2133 = 133.33 x 16
  • 2666 = 133.33 x 20

单位

1 KHz = 1 000 Hz
1 MHz = 1 000 000 Hz
1 GHz = 1 000 000 000 Hz （10亿赫兹）

超频

前置条件：

  1. CPU不锁频率
  2. 主板具备超频功能
  3. 更好的主板电源散热（非决定因素）

注：一般超频调整的是倍频，而不是外频。不稳定因素仅局限在CPU内。

显卡频率和主板基础频率

结论：显卡频率和主板基础频率是没有直接关系的。

显卡是通过PCIE通道与CPU直连 ，而PCIE是接在PCIE的控制器上（也就是北桥，现代北桥已集成到CPU内）。

总线

总线（Bus）是计算机各种功能部件之间传送信息的公共通信干线，它是由导线组成的传输线束， 按照计算机所传输的信息种类，计算机的总线可以划分为数据总线、地址总线和控制总线，分别用来传输数据、数据地址和控制信号。

关于并行与串行

  • 并行传输：字符编码的各位（比特）同时传输，也就是使用多根并行的数据线一次同时传输多个比特。
  • 串行传输：将组成字符的各位串行依次地传输，使用一根数据线传输数据，一次传输1个比特，多个比特需要一个接一个依次传输；在串行传输中又分为同步传输和异步传输。

并行总线需要兼顾数据协同传输，因此频率不能做的很高。（例如霰弹枪一次性能射出很多子弹，但是射速不高）另外并行总线，链路越多，路与路之间需要做好抗干扰，否则传输过程中数据会发生异常。

常见并行接口：

  1. PATA，Parallel Advanced Technology Attachment，并行高级技术附件接口，可用于连接硬盘、光驱、连接打印机、扫描仪等。
  2. 并口，Parallel Port，
  3. PCI，Peripheral Component Interconnect，外设部件互连接口，用于插接外置网卡、声卡、显卡和调制解调器卡等。

常见的串行接口：

  1. SATA，Serial ATA接口
  2. USB接口
  3. PCI Express，PCI E接口 （可以做多链路传输）
  4. COM串行口
