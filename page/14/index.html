  • trove 将数据库部署到openstack的虚拟机上
  • tempest 单元测试框架
  • opendaylight 开源的SDN控制器
  • ladp 轻量型目录访问协议 ， 与keystone整合 ， 实现多级用户的鉴权

毕业设计（二）

前言

眼看就要到毕业设计提交的日子了，而完成进度只有百分之五，相当于是没做。进行一轮深刻的反思后，决定对毕业设计进行重新设计，现在有一个新的思绪。之前的方案是只用单片机和语音模块之间的互动，但奈何自己才学尚浅还无法完全开发这个语音模块，目前也只是简单应用而已，并且这样的应用还不足以满足我的个人需求。于是乎，如今想出第二套方案，借助 Android 系统的简便性 ， 以及更多的运算资源 ， 决定采取安卓手机 + 51单片机 来实现这个所谓的 “智能语音机器狗” 项目。

正文

第二套方案中，分为以下三个部分 ：

  • 安卓手机 （主要用于跟人之间的交互）
  • 单片机部分 （作串口通讯，与其他电器交互）
  • 电源部分 （为整个系统提供动力，以我目前的电路设计水平，顶多也只是为单片机系统供电。）

而它能实现的功能 ：

  • 互动式报时 （人：小明同学，现在几点了？ 机 ： 现在是 上午 / 下午 , A 时 B分 ）
  • 控制其他电器的开关 （人：小明同学，关灯。 机 ： 马上关灯。）
  • 音乐播放器 （人：小明同学，播放音乐。 机 ： 收到。 ）

目前只实现这点功能，手机跟单片机之前的通讯采用蓝牙的方式，因为手机都是自带蓝牙的，所以对于这一部分的学习成本以及改造成本都不高。

对于单片机，无非是把第10号管脚(P3.0)和第11号管脚（P3.1）加装一个蓝牙模块，用以跟其他设备通讯，然而也会有这样的问题：

蓝牙的RXD和TXD会 和 语音模块的RXD、TXD冲突吗？

在单片机上要做好串口通讯逻辑的处理即可，难点在于 “中断” 。

差点忘了，还有控制动力的信号口，总共需要用到两个I/O口。

因此，整个单片机将会被用掉 4个 I/O 口，是的。。。才4个，显得很浪费。（一共32个I/O口)

对于手机，更多的是要实现如何跟人交互的任务。主要任务落在，展现给人看的界面，以及他要处理一些应答的指令。

对于界面，我有点想仿造 “天猫精灵火眼” 这款产品的界面 ，非常的讨喜，而且总体风格我也很喜欢，总结一句话来说，就是很有灵性。

利用前置摄像头，让人觉得给整个机器增加了一点点的计算机视觉的错觉。

我最喜欢的功能，“天猫精灵，现在几点了？”

界面看起来非常可爱。

就这个APP设计来说，我是非常喜欢的，讨喜的配色，猫的形象非常有灵性，动画表现起来不尴尬，也没有卡顿感，这是非常好的地方，这也是我想要去模仿的原因。然而，看了这个系统的实际测评，明显感觉到延迟，这是他语音交互做的非常不好的方法，足以抹杀掉这49块的性价比。（扯远了，我只要这个界面。）

我的实现思路是：

  • 静置 ：让眼睛部分在屏幕中，产生上下几毫米的偏移，并且会有微微缩放的效果，模拟一种呼吸的效果。
  • 执行指令时 ：对应出现相关的动画，如开灯时，让眼睛放光。报时时，让整个屏幕显示时钟。
  • 休眠时： 眼睛闭上，显示时间。

至于代码逻辑，还没有系统的思考，暂时不贴上，今天主要任务时记录这几个想法。

U盘设置 Exfat / NTFS + FAT32( PE + OS_Install)

前言

手上有太多U盘了,现在容量基本都是16G,但是我身为一个IT人士(修电脑的),随身肯定会带着各种系统的启动盘或者PE系统.然而在这么一个过程中,如果做成PE系统,目前我所测试的软件PE制作工具(大白菜,老毛桃,微PE等),把U盘制作为启动盘都会将U盘转化为FAT32的格式,在这种格式下,单个文件不能超过4G.因此,本文主要是让U盘既能正常存储文件,也能作为启动盘.

FAT32分区内无法存放大于4GB的单个文件，且性能不佳，易产生磁盘碎片。

制作启动盘

先是将ISO文件写入U盘中,例如 Windows10.iso 写入U盘 , 然后用 Diskgenius 将U盘备份到镜像文件,默认后缀名字是pmf .

Diskgenius重新给U盘分区

  • Diskgenius删除整个U盘分区.
  • 先建立第一个分区(ExFat或NTFS),再建立第二个分区(略大于一个启动盘镜像文件即可,FAT32)
  • 用 Diskgenius 将先前备份的pmf文件恢复到U盘的第二个分区.

Ubuntu中使用ExFat

UEFI与GUID下的双系统引导

前言

已经好多个星期没有写博文了,今天借着在公司实习过程中学习到的新技能,在这里秀一波. 事情是这样的,现在很多人都会去装双系统(Linux+Windows居多),如果采用Legacy的启动方式,则不适用于本教材,对于这种启动方式,自然有相对应的引导方法,下次再记录.本文记录的是UEFI+GUID模式的引导.

设置BIOS与转换硬盘分区表为GUID

首先是要对BIOS进行设置,在BIOS启动引导项内,选择”UEFI”模式.

如果硬盘分区列表不是GUID(GPT),则需要将硬盘的分区列表整体转化为GUID,这个需要在PE内,使用Diskgenius这个软件来实现,转换前,需要把所有数据做好备份,因为需要删除整个分区列表,这个是没办法的事情.

硬盘空间安排

  • 100 G 作为windows10 的系统盘(包含ESP和MSR)
  • 100 G 作为Linux的系统盘
  • 300 G 作为数据盘

安装Windows10系统

自行分配好硬盘空间,安装windows10的时候,会自动划分ESP分区(必要)和MSR(非必要),安装就没有什么好说的了.

安装Linux系统

在100G的空间内,做一下安排:

  • 200MB 的 /boot 节点
  • 两倍于物理内存的 swap 分区
  • 剩余空间分配给 / 节点

制作引导

本文使用的是rEFInd 这个开源的软件(官网:http://www.rodsbooks.com/refind/).

下载地址:http://www.rodsbooks.com/refind/getting.html

目前我只试验过在Linux下执行安装脚本后,直接运行即可.

结语

写到文章最后个人感觉难点在于 UEFI+GPT 这种方式的理解上面 , 另一方面 , 如果之前使用的是MBR,并且存放着大量的资料(几百个G的资料 - - ),这时候就要迁移资料再转变分区列表格式 . 另外 rEFInd 这个开源的引导软件,真的可以做到傻瓜式操作…在Linux下直接运行安装脚本即可,其实后续还有很多可玩的地方,比如…自定义界面??(这个后面再试试)现在是大年三十的凌晨啦,那么先晚安啦~

因为学校的网络是校园天翼宽带类型(仅限正常使用1台PC+1台手机),然而我却拥有三台笔记本,多台移动端设备,因为笔记本有时候在做Linux实验,需要连接外网下载一些安装包,而移动端有时候又想下一点奇奇怪怪的应用…总之就是有时候会想到使用网络,如果仅靠天翼宽带的账户是不可能的.另一方面,这个该死的账户,不仅限制使用的客户端,有时候还很慢,严重影响体验.

我手上有多张手机卡,里面都有30G的流量,因此,我就想利用手机的4G流量共享出来给其他设备使用.但! 我的另一个需求出现了,就是我需要所有设备都在局域网内,因为有时候需要两台设备联通共享数据或者有一台电脑作为云盘服务器,需要从里面下载或者上传资料.

总结需求:

  1. 所有设备都能上网.
  2. 所有设备都在一个局域网.

那么方案也想到了两个:

  1. 手机开放热点,所有设备连接一个手机热点,因此所有设备都在一个局域网内.
  2. 一台手机把4G网络共享给一个代理服务器,这个代理服务器一个网口连接局域网.其他设备都设置代理地址为这台代理服务器的即可.

分析:

  • 方案一,比较考验手机的性能,我的手机是一台2016年使用的手机,如今有点小毛病,如果所有设备都连接这个手机的话,可能机器会受不了,因此不采用这个方案.
  • 方案二,需要手机共享网络给代理服务器,代理服务设置好相关的路由(外网IP,内网IP),其他设备设置好网络代理即可.

开始

理论:什么是代理服务器

代理服务器（Proxy Server）是一种重要的服务器安全功能，它的工作主要在开放系统互联(OSI)模型的会话层，从而起到防火墙的作用。代理服务器大多被用来连接INTERNET（国际互联网）和INTRANET（局域网）

网络代理分类:

  1. 在线代理(在线网页代理(Web Proxy Server)，是目前简单、高速、有效的访问国际网站的最佳途径，它的功能就是用户与Web服务器之间的一个中转站，当用户访问国际网站时候，通过代理服务器来访问目标网站，并缓存在代理服务器。这样一来如果当用户访问的站点之前有人访问过，用户将直接从代理服务器上读取信息，因此显著提高浏览速度与效率。另外，网页代理还能屏蔽恶意网页代码保护您的浏览器不受干扰。)
  2. HTTP代理 (同上利用HTTP协议通讯的方式，HTTP协议即超文本传输协议，是Internet上进行信息传输时使用最为广泛的一种非常简单的通信协议。部分局域网对协议进行了限制，只允许用户通过HTTP协议访问外部网站。)
  3. SOCKET4代理
  4. SOCKET5代理(被代理端与代理服务器通过“SOCK4/5代理协议”进行通迅（具体协议内容可查看RFC文档）。SOCK4代理协议可以说是对HTTP代理协议的加强，它不仅是对HTTP协议进行代理，而是对所有向外的连接进行代理，是没有协议限制的。也就是说，只要你向外连接，它就给你代理，并不管你用的是什么协议，极大的弥补了HTTP代理协议的不足，使得很多在HTTP代理情况下无法使用的网络软件都可以使用了。（例如：OICQ、MSN等软件）SOCK5代理协议又对前一版进行了修改，增加了支持UDP代理及身份验证的功能。它不是“协议代理”，所以它会对所有的连接进行代理，而不管用的是什么协议。)
  5. VPN代理

理论: 用什么软件做代理服务器

一.Squid

Squid 是一个高性能、开源的代理缓存服务器和 Web 缓存进程，支持 FTP、Internet Gopher、HTTPS 和 SSL 等多种协议。它通过一个非阻塞的、I/O 事件驱动的单一进程处理所有的 IPV4 或 IPV6 协议请求。

Squid 由一个主服务程序 squid，和 DNS 查询程序 dnsserver，另外还有一些可选的请求重写、执行认证程序组件，及一些管理和客户端工具构成。

Squid 提供了丰富的访问控制、认证和日志环境， 用于开发 web 代理和内容服务网站应用。

二.Privoxy

Privoxy (Privacy Enhancing Proxy) 是一个非缓存类 Web 代理软件，它自带的高级过滤功能可以用来增强隐私保护、修改页面内容和 HTTP 头部信息、访问控制，以及去除广告和其它招人反感的互联网垃圾。Privoxy 的配置非常灵活，能充分定制已满足各种各样的需求和偏好。它支持单机和多用户网络两种模式。

Privoxy 使用 action 规则来处理浏览器和远程站点间的数据流。

三.Varnish Cache

Varnish Cache 是一个为性能和灵活性而生的 web 加速器。它新颖的架构设计能带来显著的性能提升。根据你的架构，通常情况下它能加速响应速度300-1000倍。Varnish 将页面存储到内存，这样 web 服务器就无需重复地创建相同的页面，只需要在页面发生变化后重新生成。页面内容直接从内存中访问，当然比其他方式更快。

此外 Varnish 能大大提升响应 web 页面的速度，用在任何应用服务器上都能使网站访问速度大幅度地提升。

按经验，Varnish Cache 比较经济的配置是1-16GB内存+ SSD 固态硬盘。

四.Polipo

Polipo 是一个开源的 HTTP 缓存代理，只需要非常低的资源开销。

它监听来自浏览器的 web 页面请求，转发到 web 服务器，然后将服务器的响应转发到浏览器。在此过程中，它能优化和整形网络流量。从本质来讲 Polipo 与 WWWOFFLE 很相似，但其实现技术更接近于 Squid。

Polipo 最开始的目标是作为一个兼容 HTTP/1.1 的代理，理论它能在任何兼容 HTTP/1.1 或更早的 HTTP/1.0 的站点上运行。

五.Tinyproxy

Tinyproxy 是一个轻量级的开源 web 代理守护进程，其设计目标是快而小。它适用于需要完整 HTTP 代理特性，但系统资源又不足以运行大型代理的场景，比如嵌入式部署。

Tinyproxy 对小规模网络非常有用，这样的场合下大型代理会使系统资源紧张，或有安全风险。Tinyproxy 的一个关键特性是其缓冲连接的理念。从效果上看， Tinyproxy 对服务器的响应进行了高速缓冲，然后按照客户端能够处理的最高速度进行响应。该特性极大的降低了网络延滞带来的问题。

客户端配置

Windows中设置代理服务器:

（IE5.0以上版本中设置代理：菜单栏“工具”->下拉菜单“Internet选项”->选项卡“连接”->在“局域网设置”中选中您目前 使用的连接，然后点击右侧的“设置”->在中间的“代理服务器”栏选中“使用代理服务器”->在“地址”和“端口”栏输入本站提供的HTTP代理服务器->确定->确定。

Linux端配置:

如果是浏览器的话,一般在设置-高级-代理,里面填写上相关的地址与端口即可.
